IE8自动开启兼容性视图

有的时候在IE8下打开浏览器访问页面的时候会莫名其妙地跳转到IE8兼容性视图下。

我到目前为止遇到过两次这个事情，留个记录作为以后解决问题的参考。


场景：页面中有一个span（行内元素）嵌套了一个div（块级元素）

原因： 结构嵌套问题

解决方案：合理的页面嵌套。不要在行内元素中嵌套块级元素，如果非要这样使用，设置行内元素的display属性为inline-block


场景：我们有一个树结构，我们给容器加了样式overflow-y: scroll，当树展开的时候，出现横向滚动条的时候，IE8切换到兼容模式

原因： 样式错误

解决方案： 将容器的样式设置为overflow: auto，怀疑可能是因为没有设置x轴的处理方式，当x轴出现滚动条时候出现异常

ztree入门使用

1. ztree官网

ztree官网地址

http://www.ztree.me/v3/main.php#_zTreeInfo

2. ztree使用

1. 引入ztree的css样式，可以根据自己的需要进行调整
2. 引入js文件，包含如下文件，根据需要进行加载

核心包（core 必须加载）

扩展包

excheck(处理checkbox和radio)

exedit(处理节点修改操作)

exhide(处理节点的隐藏和显示)

3. html结构
<ul class="ztree ztree-wrap" id="departmentTree">
</ul>

  • 由于默认的树结构是用li和ul配合生成的，所以页面中用ul作为容器

  • id是必须的，因为ztree用id来识别一棵树

4. 初始化树

我们使用的是简单的数据结构

var settings = {
  data: {
    key: {
      name: "deptName"  // 树节点显示名称对应的key
    },
    simpleData: {
      enable: true,       // 启用简单的数据结构
      idKey: "deptId",    // 节点key
      pIdKey: "deptPId",  // 父节点key
      rootPId: 0          // 根节点父key的值
    }
  }
}

var departments = [
  {deptName: xx公司, deptId: -1, deptPId: 0},
  {deptName: 研发部, deptId: 1, deptPId: -1},
  {deptName: 行政部, deptId: 2, deptPId: -1}
]

$.fn.zTree.init($('#departmentTree'), settings, departments)
5. 展开节点

节点展开处理

var treeObj = $.fn.zTree.getZTreeObj('departmentTree');
var curNode = treeObj.getNodeByParam('deptId', 1);
treeObj.expandNode(curNode, true);

这样处理有一个问题，如果当前节点是叶子节点的时候，展开功能不能focus到当前节点

我们使用下面的处理方式解决当前问题

var treeObj = $.fn.zTree.getZTreeObj('departmentTree');
var curNode = treeObj.getNodeByParam('deptId', 1);
if (curNode.isParent) {
    var currId = curNode.tId;
    curNode = curNode.getParentNode();
    treeObj.expandNode(curNode, true);
    $('#' + currId).focus();
}
else {
    treeObj.expandNode(curNode, true);
}

在部署的过程出现过下面的问题

  • 问题一：

错误信息：Deployer not found: github

解决方案：

安装插件，执行命令npm install hexo-deployer-git --save

将deploy配置中的type由github改成git

  • 问题二：

错误信息：spawn git enoen

解决方案：

配置环境变量，将git配置到path中

  • 问题三：

错误信息：Not a git repository

解决方案：

执行命令git init

  • 问题四：

错误信息： fatal: Could not read from remote repository.Please make sure you have the correct access rights.and the repository exists.

解决方案：

最开始deploy配置中的repository写成

git@github.com:yourname/yourname.github.io.git

后来改成 https://github.com/yourname/repositoryName 就可以了
